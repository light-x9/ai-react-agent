- 也可直接运行项目根目录的 `start-backend.bat`

### 重要问题：SERVER__PORT 环境变量冲突
- WorkBuddy 自身会设置环境变量 `SERVER__PORT=5933`（用于 MCP 服务端口）
- Spring Boot 的 relaxed binding 规则会将 `SERVER__PORT`（双下划线）映射为 `server.port`，覆盖 application.yml 中的 8123
- **解决方案**：启动后端时必须显式覆盖 `$env:SERVER__PORT="8123"`
- 这个问题每次在 WorkBuddy 环境中启动 Spring Boot 项目都会遇到

### 配置文件位置
- 后端主配置：`backend/src/main/resources/application.yml`（端口 8123，profile=local）
- 本地配置：`backend/src/main/resources/application-local.yml`（数据库连接、API Keys）
- 前端配置：`frontend/vite.config.js`（端口 3005，strictPort=true）
